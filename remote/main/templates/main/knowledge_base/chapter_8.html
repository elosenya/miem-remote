Глава 8. Протоколы Telnet и SSH.

Telnet — это сокращение от Terminals NETwork, протокол виртуального терминала. Обычно используется для возможности установки соединения с удаленным устройством для администрирования. Словом telnet часто называют утилиты для клиентской части. Протокол работает на прикладном уровне OSI, порт 23. Описан в RFC854.

Нет четкого понятия клиент-сервер, протокол симметричен, хотя обычно есть сервер для подключения клиентов и сами клиенты. Чаще всего используется для управления сетевыми устройствами (коммутаторы, телефонные станции и др.), а также для подключения к различным операционным системам (windows, linux и прочим) для их удаленного администрирования. Также с помощью telnet можно оперировать ftp-сессиями, управлять работой протоколов электронной почты и другими. С помощью telnet можно даже, например, подключиться к http-серверу (по порту 80, без шифрования) и получить данные веб-страницы. Именно этот протокол используется в бэкенде нашей виртуальной лаборатории.

В стандартном протоколе не предусмотрено шифрования, проверки подлинности, т.е. протокол довольно беззащитен. Для обеспечения безопасности telnet стоит применять только в защищенных сетях или использовать иные протоколы со встроенной в них защитой (например SSH). Но при этом чаще всего именно telnet является включенной по умолчанию опцией для управления коммутаторами и прочими сетевыми устройствами

С помощью telnet можно даже передавать такие нестандартные вещи, как ASCII анимацию. В этом можно убедиться, набрав команду: telnet towel.blinkenlights.nl (стоит учесть, что в ОС Windows telnet-клиент по умолчанию выключен, необходимо его установить через установку и удаление компонентов Windows или воспользоваться альтернативами, например Putty).

В telnet по умолчанию используется только символы таблицы ASCII и стандартные опции, но системы могут «договориться» передавать данные в бинарном режиме и включать дополнительные опции. Варианты ввода: построчный и посимвольный.

SSH — Secure SHell. Как видно из названия — в отличие от telnet разрабатывался изначально как защищенный. Шифруется весь трафик. Используется для удаленного администрирования, передачи файлов, создания туннелируемых соединений. Протокол работает на прикладном уровне OSI, порт 22.

Варианты аутентификации:

  1. По паролю: при подключении генерируется общий ключ
  2. По ключевой паре: ключевая пара сгенерирована заранее и распространена на клиент и сервер
  3. По IP-адресу Используется алгоритм RSA/DSA. Есть возможность использовать сжатия данных. В SSH (v2) в отличие от протокола telnet, подключиться к сессии и считать данные невозможно.

Для работы SSH необходим сервер и клиент. Обычно используется по умолчанию для управления *NIX (linux и прочими) системами. Для проверки работы SSH можно установить (или запустить) любой linux дистрибутив (например ubuntu) в виртуальной машине, а затем подключиться к нему через SSH клиент, например Putty. Возможно, что для SSH подключения в процессе установки системы нужно будет выбрать опцию OpenSSH Server. Логин и пароль для подключения — это тот пользователь, который входит и в локальную систему.

Протокол SSH можно использовать и для передачи файлов (SCP, secure copy). Для этого можно использовать как консоль linux, так и графические приложения, например WinSCP. Кроме передачи файлов, с ними можно выполнять различные действия.

Протокол SSH можно также использовать для туннелирования соединения, почти как VPN. Данные, передаваемые в этом туннеле будет зашифрованы и недоступны для окружающих. Причем такие соединения можно перенаправлять через любые порты, например 443, если где-то запрещено использование других портов (например публичных WiFi точках).
